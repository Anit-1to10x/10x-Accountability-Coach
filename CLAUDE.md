# 10X Accountability Coach - Claude Code Instructions

## IMPORTANT: First-Time Setup Detection

**Before running ANY commands**, Claude Code MUST check if setup has been completed by reading `.setup-log.json` in the project root.

### Setup Log Check Flow
```
1. Read .setup-log.json using Read tool
2. If setupComplete === false OR file doesn't exist:
   → Run FULL SETUP FLOW (Step 1-9)
3. If setupComplete === true:
   → Run RETURNING USER FLOW (just start the app)
```

---

## FULL SETUP FLOW (First Time / Fresh Clone)

When a user clones this repo and says **"set up"**, **"install"**, **"start my app"**, **"clone and set up"**, or similar:

### Step 1: Detect Operating System

**Run this command first:**
```bash
node -e "console.log(JSON.stringify({platform: process.platform, arch: process.arch, nodeVersion: process.version}))"
```

Results:
- `win32` = Windows
- `darwin` = Mac
- `linux` = Linux

### Step 2: Check Setup Log

**Read `.setup-log.json` using the Read tool:**
- If `setupComplete: true` → Skip to **RETURNING USER FLOW**
- If `setupComplete: false` or file missing → Continue with setup

### Step 3: Ask Data Source Preference

**IMPORTANT: Ask the user this question:**

> **How would you like to store your data?**
>
> 1. **Local Files (Recommended)** - Data stored in `./data` folder. Best for personal use, no external setup needed.
>
> 2. **Supabase Cloud Database** - For Vercel deployment or multi-device sync. Requires Supabase account.

**If user chooses Local:**
- Set `DATA_SOURCE=local` in configuration
- No additional setup needed

**If user chooses Supabase:**
- Ask: "Please provide your Supabase Project URL (e.g., https://xxxxx.supabase.co):"
- Ask: "Please provide your Supabase Anon Key:"
- Set `DATA_SOURCE=supabase` in configuration
- Remind user to run `data/schemas/full_schema.sql` in Supabase SQL Editor

### Step 4: Ask for API Key

**Ask the user:**

> **Do you have an OpenAnalyst API key?**
>
> Get one free at: **https://10x.events/api-key**
>
> Please paste your API key (starts with `sk-oa-v1-`):

**Validate the key format** - must start with `sk-oa-v1-`

### Step 5: Install Dependencies

**Run these cross-platform commands:**

```bash
# Install root dependencies
npm install

# Install UI dependencies
cd ui && npm install
```

**Record versions after install:**
```bash
node -e "console.log('Node:', process.version); const {execSync} = require('child_process'); console.log('NPM:', execSync('npm --version').toString().trim())"
```

### Step 6: Create Environment File

**Create `ui/.env.local` using the Write tool:**

```env
# ==============================================================
# 10X ACCOUNTABILITY COACH - ENVIRONMENT CONFIGURATION
# ==============================================================
# Generated by Claude Code Setup on <CURRENT_DATE>
# ==============================================================

# ==============================================================
# REQUIRED: OpenAnalyst AI API
# ==============================================================
OPENANALYST_API_URL=https://api.openanalyst.com/api
OPENANALYST_API_KEY=<USER_PROVIDED_API_KEY>
OPENANALYST_MODEL=openanalyst-beta

# ==============================================================
# DATA SOURCE CONFIGURATION
# ==============================================================
DATA_SOURCE=<local_or_supabase>

# ==============================================================
# SUPABASE (Only required if DATA_SOURCE=supabase)
# ==============================================================
SUPABASE_URL=<SUPABASE_URL_OR_LEAVE_EMPTY>
SUPABASE_ANON_KEY=<SUPABASE_KEY_OR_LEAVE_EMPTY>
```

### Step 7: Update Setup Log

**Update `.setup-log.json` with complete installation details:**

```json
{
  "setupComplete": true,
  "setupDate": "<ISO_TIMESTAMP>",
  "platform": "<win32|darwin|linux>",
  "nodeVersion": "<version>",
  "npmVersion": "<version>",
  "dataSource": "<local|supabase>",
  "dependencies": {
    "root": { "installed": true, "installedAt": "<timestamp>" },
    "ui": { "installed": true, "installedAt": "<timestamp>" }
  },
  "configuration": {
    "apiKeyConfigured": true,
    "supabaseConfigured": <true_or_false>,
    "envFileCreated": true
  },
  "firstRun": {
    "profileCreated": false,
    "onboardingComplete": false
  },
  "logs": ["<timestamp> - Setup completed successfully"]
}
```

### Step 8: Start the Application

```bash
# Kill any existing process on port 3000
npx kill-port 3000

# Start the development server
cd ui && npm run dev
```

### Step 9: Guide User to Onboarding

**Tell the user:**

> **Setup Complete!**
>
> Your 10X Accountability Coach is running at **http://localhost:3000**
>
> Since this is your first time, you'll be guided through:
> 1. Creating your profile
> 2. Quick onboarding to personalize your experience
> 3. Starting your first accountability challenge
>
> Open http://localhost:3000 to begin!

## CROSS-PLATFORM COMMANDS

Always use these cross-platform commands:

| Task | Command |
|------|---------|
| Install deps | `npm install` |
| Kill port | `npx kill-port 3000` |
| Delete files | `npx rimraf <path>` |
| Read file | Use Read tool (NOT cat) |
| Write file | Use Write tool (NOT echo) |
| Check file exists | Use Glob tool |

**NEVER use platform-specific commands like:**
- `rm -rf` (Unix only)
- `del /q` (Windows only)
- `cat` (Unix only)

## DATA SOURCE CONFIGURATION

### Local Mode (Default)
- All data stored in `./data` folder
- Profile data in `data/profiles/{profile-id}/`
- No internet required for data operations
- Best for personal/local use

### Supabase Mode
- All data synced to Supabase cloud
- Required env vars: `SUPABASE_URL`, `SUPABASE_ANON_KEY`
- User must run `data/schemas/full_schema.sql` in Supabase
- Best for Vercel deployment or multi-device sync

### Switching Data Sources
1. Update `DATA_SOURCE` in `ui/.env.local`
2. If switching to Supabase, add credentials
3. Restart the app: `npx kill-port 3000 && cd ui && npm run dev`

## TROUBLESHOOTING

### "API Disconnected" in Settings
1. Check `ui/.env.local` has valid API key (not placeholder)
2. Restart: `npx kill-port 3000 && cd ui && npm run dev`

### Port Already in Use
```bash
npx kill-port 3000 3001 3002
cd ui && npm run dev
```

### Dependencies Issues
```bash
npx rimraf node_modules ui/node_modules
npm install && cd ui && npm install
```

### Setup Not Working
```bash
# Delete setup log and try again
npx rimraf .setup-log.json
# Then say "start my app"
```

### Build Errors
```bash
cd ui && npx rimraf .next && npm run build
```
